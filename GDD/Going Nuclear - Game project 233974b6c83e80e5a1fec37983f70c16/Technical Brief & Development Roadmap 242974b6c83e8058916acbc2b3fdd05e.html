Technical Brief & Development Roadmap

Version: 1.2
Date: July 17, 2025
Document Purpose: This document serves as a technical overview and a statement of work for new developers joining the project. It should be read in conjunction with the main Game Design Document (GDD).

Part 1: The Vision (GDD Core Concepts)

  • High Concept: A turn-based, satirical grand strategy game inspired by the 1989 classic "Nuclear War," rebuilt in Godot with a 3D interactive console interface.
  • Core Experience: Players, acting as a neutral "hidden hand," manage resources (AP, Treasury) to acquire and play cards. The goal is to manipulate a world of 4 AI-controlled, caricatured faction leaders to achieve victory, either through total annihilation or a difficult diplomatic path.
  • Game Flow: The game is structured in two phases.
    1. Phase 1: Deterrence & Escalation (Cold War): Focus on Espionage, InfoWar, and building alliances. Direct military attacks are heavily penalized.
    1. Phase 2: Kinetic & Annihilation (Hot War): Triggered by low DEFCON status. Military conflict becomes the primary focus.
  • Aesthetic: A tactile, retro-futuristic "war room" feel, with all interactions happening through a physical 3D console. The tone is dark humor and satire.

Part 2: Successfully Implemented Systems (As of This Document)

The following systems are implemented, functional, and form the stable core of the game.

1. Core Game Engine & Turn Management (GameManager.gd)

  • Turn Loop: The game correctly cycles through a 5-player turn order: 1 Human ("Null Faction") and 4 AI Factions.
  • Resource System: Action Points (AP), Treasury, and Morale are implemented. AP and Treasury are correctly generated for all players at the start of their turns based on faction stats.
  • Player Model: The "Null Faction" model is implemented. The human player is a distinct entity with their own resource pool and hand, separate from the AI factions.

2. Data Architecture (Modular & Data-Driven)

  • "Smart Resources": Both FactionData.gd and CardData.gd have been implemented as @tool scripts. This allows for rapid creation of new factions and cards directly in the Godot editor via a "Preset" dropdown menu, which auto-populates all data from a centralized database within the script.
  • Global Managers (Autoloads): FactionDatabase, CardDatabase, and Logger are implemented as singletons, providing robust, global access to game data and logging functionality.

3. 3D Interactive Console (The Frontend)

  • 3D Scene: The game is rendered in a 3D scene (Console3D.tscn) which contains the main console model and lighting.
  • 3D Button Interaction: A modular button system (PhysicalButton3D.gd) is in place.
    • 3D meshes with an attached StaticBody3D and CollisionShape3D are interactive.
    • A raycasting script (InteractionController.gd) detects clicks on these 3D objects.
    • The main_controller.gd script correctly receives signals from 3D button clicks and triggers game logic.
  • 3D Button Animation: The AnimationPlayer is functional. 3D buttons correctly play a "press" animation when clicked. The "stay depressed" logic for target selection is also implemented.
  • Control Disabling: 3D button collision shapes are correctly disabled during AI turns, preventing player input.

4. Core Player Actions

  • Build Action: The "Build" button is functional. Clicking it correctly costs 1 AP and a set amount of Treasury, and adds two cards to the player's hand.
  • Card Selection UI: A pop-up DeploymentScreen UI appears when a card category button is clicked, correctly displaying the available cards from the player's hand. It also features a "Card Visualizer" that shows card details on hover.
  • Target Selection: Clicking the 3D enemy avatar buttons correctly selects them as a target in the GameManager, with visual feedback provided by the "stay depressed" animation.

Part 3: Systems to Be Implemented (The Roadmap)

The following features have been designed in the GDD but are not yet implemented in the code.

1. 3D Integrated Information Screens (Immediate Next Step)

  • Task: Finalize the implementation of the SubViewport system to render dynamic information (Game Log, Enemy Stats, Player Stats) onto the 3D screens of the console model.
    • Sub-task: Style the 2D source scenes (LogScreen2D, InfoScreen2D) to match the desired "LCD" aesthetic.
    • Sub-task: Ensure the Logger is correctly connected to the 3D log screen.

2. Complete the Action Logic

  • Task: Wire up the Delivery, Payload, InfoWar, Utility, and Defense 3D buttons to their corresponding game logic.
    • The _on_action_button_pressed function needs to be fully implemented in main_controller.gd.
    • The process_player_attack function in GameManager needs to be finalized to handle all attack types.

3. Advanced AI & Faction Asymmetry

  • Task: Expand the AIController to fully utilize all personality_traits from the GDD (e.g., PATIENT AI saves cards, AGGRESSIVE AI prioritizes attacks).
  • Task: Implement the unique faction abilities (e.g., USA's "Sanctions," North Korea's "Superweapon"). This will likely require adding new functions to the GameManager and new Action Buttons for the relevant factions.

4. Status Effects & Damage Over Time

  • Task: Implement the status_effects array in the PlayerState.
  • Task: Create the logic in the GameManager's start_turn function to process active effects (like "Fallout" from a Dirty Bomb).

5. Global Systems

  • Task: Implement the global DEFCON variable. This involves adding logic to the GameManager to track it and functions to apply its global penalties when it drops to certain levels.
  • Task: Implement the "World Events" system by creating a deck of Event Cards and having the GameManager draw and resolve one at the start of each full round.

6. Victory Conditions

  • Task: Create a WinConditionManager.gd (which you have already created a placeholder for) that checks for the Annihilation Victory (only one player left) at the end of each turn.
  • Task: Implement the complex "Diplomatic Victory" action, including the alliance requirements and the AI voting logic.

This document provides a complete snapshot of where we are and a clear, feature-based roadmap of where we are going.
